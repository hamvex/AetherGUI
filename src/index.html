01

Introduction to Aether

Aether is a networking tool that discovers usable Cloudflare endpoints and creates a local SOCKS5 proxy through MASQUE, WireGuard, or gool. Firstham AetherGui is a Windows frontend for the official Aether core. It does not reimplement tunneling or scanning; it starts the bundled core safely, passes supported settings, displays its logs, and stops it cleanly.

Important

Aether creates a proxy, not a system-wide VPN by itself. Applications must be configured to use 127.0.0.1:1819, or routed through a tool such as Proxifier or v2rayN.

02

How Aether works

When you press Connect, the GUI validates your settings and starts the official Aether core as a hidden child process. Aether scans for a healthy endpoint, establishes the selected tunnel, then listens locally for SOCKS5 traffic.

Application sections and indicators

Shows state, endpoint, SOCKS5 address, elapsed time, and primary actions.

Contains custom listener, endpoint, keepalive, watchdog, timeouts, and config paths.

Live logs

Shows core output. Copy it when reporting a problem; Clear only clears the display.

Verify connection

Sends an HTTPS request through the configured SOCKS5 proxy to Cloudflare trace.

System tray

Closing the window hides it. Use the tray menu to Connect, Disconnect, Show, or Exit.

Connection states

  • Disconnected — no Aether process is serving the proxy.
  • Scanning — Aether is searching for a usable endpoint.
  • Connecting — a tunnel is being negotiated.
  • Connected — the local SOCKS5 proxy is ready.
  • Reconnecting — Aether is restoring a lost tunnel.
  • Error — review the message and Live logs.
04

Complete first-time setup

  1. Install the application

    Download the latest x64 Setup from GitHub Releases, run it, and complete the installer. The MSI and portable ZIP are alternatives.

  2. Launch Firstham AetherGui

    Open it from the Start Menu. Read the welcome tour; no terminal or Rust installation is required.

  3. Keep the recommended profile

    Select MASQUE, Balanced scan, IPv4, Firewall obfuscation, and HTTP/3.

  4. Connect

    Press Connect and wait for Connected. Scanning can take time depending on the network.

  5. Verify the tunnel

    Press Verify connection. A readable Cloudflare trace result confirms that the SOCKS5 request succeeded.

  6. Configure your application

    Use SOCKS5 host 127.0.0.1 and port 1819. See the v2rayN and Proxifier guides below.

05

MASQUE, WireGuard, and gool

RECOMMENDED

MASQUE

HTTP-based tunneling. HTTP/3 is fast on healthy UDP networks; HTTP/2 provides a TCP fallback. Start here.

WireGuard

A lightweight UDP tunnel. It can perform well when WireGuard traffic is usable on the current network.

gool

Aether's WARP-in-WARP mode. It is an alternative path for networks where the other protocol profiles are unreliable.

There is no universally best protocol. Filtering and routing differ by provider, location, and time. If the recommended MASQUE profile fails, try its HTTP/2 transport before switching protocol.

07

Scan modes

Turbo

Prioritizes speed. Use on a good network when you want the quickest discovery.

Balanced

Recommended. A practical balance of discovery speed and coverage.

Thorough

Searches more widely and takes longer. Try it when Balanced finds no healthy endpoint.

Stealth

Uses a more conservative scanning pattern for sensitive networks.

IP scan: IPv4 is the safest default. Choose IPv6 only with working native IPv6. Both increases coverage but can extend scan time.

08

Obfuscation profiles

Obfuscation changes observable traffic patterns. The GUI only shows profiles valid for the selected protocol.

  • MASQUE: Firewall (recommended), GFW, Off.
  • WireGuard and gool: Balanced (recommended), Aggressive, Light, Off.

Aggressive modes can add overhead. Start with the recommended profile and only increase obfuscation when simpler profiles fail.

09

Watchdog and process safety

The GUI supervises the Aether child process. With Watchdog enabled, a connection attempt that stops making progress beyond the configured stall timeout is terminated. This prevents stuck or unmanaged background processes. Keep Watchdog enabled and use the default 90-second timeout unless your network regularly needs longer scans.

On Exit, the GUI stops its child process cleanly. Single-instance protection prevents multiple GUI copies from starting separate unmanaged cores.

Configure SOCKS5 in v2rayN

Menu names vary slightly between v2rayN releases, but the values remain the same.

  1. Connect Aether first

    Wait until Firstham AetherGui shows Connected.

  2. Add a SOCKS server

    In v2rayN, choose the option to add a SOCKS server/profile.

  3. Enter the proxy

    Remarks: Aether
    Address: 127.0.0.1
    Port: 1819
    Username/Password: empty

  4. Save and select it

    Use the profile directly, or select its alias as a Front Proxy for another v2rayN node.

  5. Avoid loops

    If v2rayN TUN is enabled, create Direct/bypass rules for aether-gui.exe and aether.exe.

Direct profileApplication→v2rayN→Aether SOCKS5→Internet
Front proxyApplication→v2rayN node→Aether front proxy→Internet

Do not make Aether both the input and output of the same route. That creates a proxy loop and stops traffic.

Configure SOCKS5 in Proxifier

  1. Add the proxy server

    Open Profile → Proxy Servers → Add.

  2. Enter Aether details

    Address: 127.0.0.1
    Port: 1819
    Protocol: SOCKS Version 5
    Authentication: Off

  3. Test it

    Press Check while Aether is Connected, then save.

  4. Create a Direct safety rule

    In Profile → Proxification Rules, add a top rule for aether-gui.exe; aether.exe with Action Direct.

  5. Create your app rule

    Add the desired application executables below the Direct rule and choose the Aether proxy as Action. Keep the Localhost rule Direct.

  6. Proxy DNS

    Under Profile → Name Resolution, enable resolving hostnames through the proxy when appropriate.

1 · Aether Directaether-gui.exe, aether.exeDIRECT
2 · Selected applicationsbrowser.exe, app.exeSOCKS5 127.0.0.1:1819
3 · DefaultEverything elseYour choice

Rule order matters: Proxifier applies the first matching rule. SOCKS5/Proxifier is primarily for TCP; UDP-only applications require a suitable TUN solution.

13

Verify traffic is passing through Aether

  1. Confirm the application state is Connected.
  2. Press Verify connection. A Cloudflare trace response confirms the GUI reached HTTPS through SOCKS5.
  3. In v2rayN or Proxifier, open its connection/log view and confirm the target application matches the SOCKS rule.
  4. Visit a reputable IP-check page from the proxied application and compare it with a direct connection.
  5. Temporarily Disconnect Aether. The proxied application should stop reaching the network if it has no direct fallback.
Do not rely on only one signal.

A changed IP does not prove DNS is proxied. Enable remote/proxy DNS and review client logs as well.

Updating the application

  1. Open the About page and select Check GitHub Releases.
  2. Compare the latest version with the version shown in the sidebar.
  3. Disconnect and exit Firstham AetherGui from the tray.
  4. Download and run the new Setup. It upgrades the current-user installation.
  5. Launch it, confirm your settings, and run Verify connection.

Only download releases from the official hamvex/AetherGUI repository. Published files are currently unsigned and may trigger SmartScreen.

15

Frequently asked questions

Is this a replacement for Aether?

No. It is a Windows frontend that bundles and supervises the official Aether core.

Is Aether a system-wide VPN?

Not by itself. It exposes a SOCKS5 proxy. Configure each app, v2rayN, Proxifier, or another compatible routing tool.

Which settings should I use first?

MASQUE, Balanced, IPv4, Firewall, and HTTP/3. If it fails, try HTTP/2, then Thorough scan.

Can I change port 1819?

Yes, in Advanced Settings. Update every client to use the same new port.

Why does closing the window not exit?

The app minimizes to the system tray so a live proxy is not interrupted. Use Tray → Exit for full shutdown.

Does the GUI store credentials?

No credentials or private keys are stored in its settings. It persists configuration choices and file paths.

16

Troubleshooting common issues

Stuck on Scanning

Try Thorough mode, IPv4, remove a custom endpoint, and allow more time. Check whether another Aether instance is running.

HTTP/3 cannot connect

Switch MASQUE transport to HTTP/2. UDP or QUIC may be blocked or throttled.

Port 1819 is already in use

Exit other Aether instances or choose another localhost port and update your proxy client.

Connected, but an app has no internet

Confirm SOCKS5—not HTTP—is selected, verify host/port, enable proxy DNS, and inspect routing rules.

Proxifier loop or repeated failures

Move a Direct rule for aether-gui.exe and aether.exe above all proxy rules.

Connection test fails

Review Live logs, check firewall/antivirus outbound permissions, Disconnect, then try HTTP/2 or another protocol.

SmartScreen warning

The current binaries are unsigned. Download only from the official Release and verify SHA-256.

Need support

Copy Live logs, remove sensitive information, and include protocol, transport, scan mode, and Windows version in the report.
